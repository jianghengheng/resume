**项目四 React 组件库开发与私有化部署**                                                                                                                                                     

- **技术栈**  React 18 + TypeScript + Storybook + Verdaccio + Vite

- **项目描述**: 司内部多个前端项目存在重复开发通用组件的问题，导致维护成本高、体验不一致。为此，我
主导设计并实现了一套 React 组件库，并通过 Verdaccio 搭建私有 npm 仓库，实现组件共享与版本管理

- **个人职责** ：

1. 组件库设计与开发
   基础组件：开发按钮、输入框、表格、弹窗等高频组件，支持主题定制与国际化。
   复杂组件：封装在线代码编辑预览组件，代码对比组件。
   文档与示例：使用 Storybook 搭建组件文档站点，支持交互式演示与代码片段复制。


    1. 私有 npm 仓库搭建
  Verdaccio 部署：在内部服务器部署 Verdaccio，配置用户权限与插件（如 verdaccio-auth-memory）。
  自动化发布：集成 GitHub Actions，实现组件库版本自动发布到私有仓库。
  依赖管理：通过 .npmrc 配置私有仓库地址，确保团队无缝接入。
    2. 工程化与性能优化
  打包构建：使用 Vite 打包组件库，支持按需加载与 Tree Shaking。
  类型支持：为所有组件提供完整的 TypeScript 类型定义，提升开发体验。
  体积优化：通过 babel-plugin-import 实现按需引入，最终打包体积减少 60%。
    3. 团队协作与规范
  代码规范：制定 ESLint + Prettier 规则，统一代码风格。
  提交规范：使用 Commitlint + Husky 规范 Git 提交信息。
  文档贡献：编写《组件开发指南》，推动团队成员参与组件库维护
